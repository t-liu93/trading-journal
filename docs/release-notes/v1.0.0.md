# v1.0.0 — Trading Journal

**The first complete release: a self-hosted, single-user trading journal for stocks, forex CFDs, and multi-leg options strategies.**

v1.0.0 is a ground-up rebuild around an *atomic-trade* data model — a generic `Position` aggregates individual trades, so adding new instruments or strategies is additive rather than a rewrite. It ships as a single Docker container that serves both the API and the web UI.

---

## ✨ Highlights

- **Every strategy, one model** — spot (stocks & forex), the wheel, iron condors, and LEAP + PMCC all run on the same position/trade core.
- **Trade-led, audit-friendly** — multi-leg entry with server-computed cash flow, order grouping, and recoverable soft-delete.
- **Analytics out of the box** — per-currency realized P&L, monthly chart, win-rate, and open/closed position tables.
- **One-container deploy** — FastAPI serves the built SPA and the API on a single port; migrations run automatically on startup.
- **Multi-arch images** — `linux/amd64` + `linux/arm64` published to GHCR.

---

## 🚀 Features

### Accounts & authentication
- Email/password registration and login with secure cookie sessions (FastAPI Users, argon2 hashing).
- Multiple trading accounts (`cash` / `margin` / `paper`) — create, edit, archive.

### Instruments
- Stocks, options, and forex pairs with type-specific fields (option `strike`/`expiry`/`type`/`multiplier`, forex base/quote).
- Typeahead picker with inline **create-on-the-fly** — get-or-create means duplicate symbols resolve to the existing instrument instead of erroring.

### Positions & strategies
- A generic **Position** aggregates atomic trades — the same surface handles every strategy type:
  `spot_stock` (long), `spot_forex` (long/short), `wheel`, `iron_condor`, and `pmcc` (LEAP + poor-man's covered call).
- Position list with strategy/status filters, plus a detail page with four tabs:
  - **Overview** — manual fields (capital used, max risk/reward, notes) and read-only derived values (days open, realized P&L, ROI, result).
  - **Meta** — wheel funding/loan/interest; PMCC LEAP selection.
  - **Plan** — append-only trade-plan revision history (entry / stop / target — for planned trades such as forex).
  - **Trades** — chronological log grouped by order, with auto-detected pattern badges (IC-open / Assignment / Exercise / Expiration).

### Trades
- Single or multi-leg entry; all legs of one submission share a server-assigned order group.
- Cash flow is **computed on the server** (source of truth); action↔instrument validation enforced.
- Audit-friendly soft-delete — archived trades are hidden but recoverable.

### Dashboard & analytics
- Per-currency realized-P&L cards, open & closed position tables, a monthly realized-P&L bar chart (ECharts), and a win-rate gauge.
- Per-position net cash flow computed in a single batched query on list/detail views.

## 📦 Deployment

- **Single-container image** — FastAPI serves the Vite-built SPA *and* the API on one port (`8000`), with SPA deep-link fallback.
- Runs `alembic upgrade head` automatically on startup.
- **SQLite** by default (schema is Postgres-ready), stored in a host-owned bind-mounted `./data` dir; the container runs as **your host user** (non-root), with a healthcheck and `restart: unless-stopped`.
- An example `docker-compose.yml` lives under [`example/`](https://github.com/t-liu93/trading-journal/tree/v1.0.0/example); it binds to loopback for SSH-tunnel setups.

### Quick start

```bash
mkdir trading-journal && cd trading-journal

# Grab the example compose
curl -O https://raw.githubusercontent.com/t-liu93/trading-journal/v1.0.0/example/docker-compose.yml

# Create the data dir (owned by you) and a .env with a real secret
mkdir -p data
echo "COOKIE_SECRET=$(python3 -c 'import secrets; print(secrets.token_urlsafe(32))')" > .env

# Pull and run
docker compose up -d
```

Then open **http://localhost:8000**. Full guide: [`docs/walkthrough.md`](https://github.com/t-liu93/trading-journal/blob/v1.0.0/docs/walkthrough.md) (English) · [`docs/walkthrough.zh.md`](https://github.com/t-liu93/trading-journal/blob/v1.0.0/docs/walkthrough.zh.md) (中文).

## 🧱 Tech stack

- **Backend** — Python 3.12 · FastAPI · SQLAlchemy 2 (async) · Alembic · FastAPI Users · Pydantic Settings · Uvicorn · SQLite (aiosqlite); managed with `uv`.
- **Frontend** — Vue 3 · Vite · TypeScript · Naive UI · Pinia · Vue Router · ECharts (`vue-echarts`); API types generated from the OpenAPI schema (`openapi-typescript`).

## ⚠️ Known limitations (planned for later releases)

- **Unrealized P&L** / live market quotes — no quote provider yet; only realized P&L is computed.
- **Cross-currency FX conversion** — each currency is reported separately (no consolidated total).
- **Broker / CSV import** — manual entry only.
- **OAuth / MFA** — password auth only.
- **Mobile-responsive layout, dark mode, i18n** — not yet.
- **Position archiving** — closed positions are filtered, not archived.
